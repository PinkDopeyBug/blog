# 在window对象上绑定数据的的题
1. 命名冲实
后绑定的重名数据系把前面的覆盖
2. 安全风险
任何人可以获取和修改。
3. 性能问题，
占用的存如果不解绑也不会清除
4. 全局污染
任何人都可以获取到也可以修改

解决：模块化、命名空间、IIFE、开启严格模式、qiankun代理沙箱

# 地址栏输入url回车后发生了什么
1. url检测：纠错、补全、转换
	检测url是否有误，www.baidu.com补全为http://www.baidu.com/，如果有非英文或字符的话还会转换为ascii码
2. DNS解析
将域名解析成IP地址，先查询用户设备上是否有域名缓存，如果没有则需要向上级DNS服务器查询
优化：使用`dns-prefetch`预解析、减少域名数量、使用第三方知名域名
